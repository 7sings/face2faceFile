# 面对面快传

一个局域网优先的跨端文件快传 Web App。两台设备连接同一个 Wi-Fi 后，通过浏览器打开页面即可互传文件，不需要安装 App。

## 技术方案

- 前端：原生 HTML/CSS/JavaScript，移动端友好。
- 信令：Node.js HTTP + WebSocket，用于房间管理和 WebRTC offer/answer/candidate 转发。
- 文件传输：WebRTC DataChannel 点对点传输，文件数据不经过服务器。
- 分片策略：File API 按 64KB 分片读取发送，接收端合并为 Blob 并生成下载链接。
- 房间机制：自动生成房间码和分享链接，另一台设备输入房间码或打开链接加入。

## 本地启动

```bash
cd /Users/zackingwu/Code/demo/face2faceFile
npm install
npm start
```

启动后终端会输出类似：

```text
本机访问: http://localhost:8090
局域网访问: http://你的局域网IP:8090
```

两台设备连接同一个 Wi-Fi，使用终端输出的“局域网访问”地址打开页面。

## 使用流程

1. 第一台设备打开页面，复制分享链接或记住房间码。
2. 第二台设备打开分享链接，或打开页面后输入房间码加入。
3. 页面显示“可以传文件”后，选择文件或拖拽文件发送。
4. 接收端在“收到的文件”区域点击下载。

## 已实现能力

- 自动创建房间码和分享链接。
- 支持另一台设备加入房间。
- 支持 WebRTC DataChannel 直连建链。
- 支持多文件选择和拖拽上传。
- 支持大文件分片传输。
- 支持发送/接收进度展示。
- 支持图片文件缩略图预览。
- 支持接收完成后生成单文件下载链接。
- 支持勾选文件后直接批量下载。
- 支持勾选图片后调起系统面板保存到相册。
- 支持单张图片长按保存兜底。
- 支持勾选文件后浏览器端打包 ZIP 下载。
- 支持连接状态和日志展示。
- 启动时自动打印局域网 IP 访问地址。

## 当前限制

- 当前版本优先支持同一局域网直连，没有接入 TURN 中继。
- 部署到公网域名时建议使用 HTTPS，避免移动端浏览器限制部分 Web API。
- 如果两台设备不在同一网络，可能需要增加 STUN/TURN 服务。
